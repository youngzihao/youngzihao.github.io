Chapter 3: Model-View-Controller Pattern

  • Model 持有应用的数据，通常是 structs 或者简单的 class

  • View 显示视觉元素，并且控制屏幕，通常是 UIView 的子类

  • Controller 协调 Models 和 Views，通常是 UIViewController 的子类

  • Controller 与 Model 和 View 是强关系型，所以 Controller 能直接获取它们，相反， Model 和 View 不应该对拥有它们的 Controller 强引用，这将会导致循环引用
    。相反，Model 和它的 Controller 通过 property observing 进行交流，View 则通过 IBActions 和 Controller 进行交流

  • 在几个 Controller 中你可以重复使用 Model 和 View，但 Controller 一般不能重复使用，因为它的逻辑是针对特定任务的

    Views 可以通过 delegate 对自己的 Controller 有一个弱引用，举一个例子，一个 UITableView 对于它的 view controller 可能有一个弱引用，因为它的 delegate 或 dataSource 引用，然而 table view 并不知道这就是它的 view controller
